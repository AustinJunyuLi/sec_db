# Research Target and Estimator Contract

The pipeline produces structured data for empirical work on takeover sale processes. The downstream estimator consumes one row per bidder-cycle and treats each auction cycle as an observation. The pipeline must therefore solve two related tasks. First, it must represent the full merger process in a durable canonical form. Second, it must produce the current bidder-cycle view from that form without manual choices at projection time.

The binding current contract is `derive_views.py`. That file expects a canonical DuckDB-like store containing at least `deals`, `actors`, `events`, `event_actor_links`, `process_cycles`, and `judgments`. The view logic constructs bidder-cycle rows by finding bidder actors linked to events in a cycle; finding the formal-boundary judgment for that cycle; ranking proposal events before the boundary to obtain `bI`, `bI_lo`, and `bI_hi`; ranking proposal events at or after the boundary to obtain `bF`; mapping `bidder_subtype` to `T`; and joining auxiliary judgments for scope validity, dropout mechanism, cycle visibility, and valuation comparability.

The bidder-cycle row fields that must be producible are: `deal_slug`, `cycle_id`, `actor_id`, `actor_label`, `bI`, `bI_lo`, `bI_hi`, `w_logwidth`, `bF`, `admitted`, `T`, `bid_value_unit`, `consideration_type`, `boundary_event_id`, `boundary_quality`, `formal_boundary`, `dropout_mechanism`, `dropout_has_alternative`, `cycle_visibility`, `scope_validity`, `valuation_comparability`, and `confidence_min`. The view computes `w_logwidth` as `log(bI_hi / bI_lo)` when positive bounds exist. It computes `admitted` as the presence of a post-boundary eligible formal proposal, not as a free-standing board decision. The canonical representation should still store admission decisions separately, because an admitted bidder can withdraw without submitting a formal bid; the current estimator view simply does not consume that distinction directly.

The cycle-level quantities required by the research target are deterministic functions of bidder rows and cycle metadata. `J` is the count of bidder rows in the cycle, subject to the analyst’s chosen scope filter. `g^S` is the fraction of rows whose `T` equals one. `r_23` is computed from the second- and third-highest comparable informal bid centers. `mean_w` is the mean of non-null `w_logwidth`. `S` is true when the cycle has a real formal stage and at least one eligible post-boundary formal proposal; it is false when the cycle is explicitly informal-only or terminated before a formal stage. If a cycle has fewer than three comparable informal bids, `r_23` is null and the valuation-comparability judgment must explain why.

The canonical representation must exceed the estimator view. It must preserve target and acquirer advisors, legal counsel, board committees, term details, go-shop windows, termination fees, reverse termination fees, financing-related events, confidentiality agreements, data-room access, management presentations, exclusivity, process terminations, restarts, alias mappings, group membership, prior relationships, and unresolved ambiguities. These objects support future projections such as whether counsel predicts boundary existence, whether go-shop length predicts topping bids, or whether board committee structure affects screening.

A cycle is not assumed to have a formal boundary. For a no-boundary cycle, the canonical store creates a boundary-absence marker event dated at the cycle end or best-supported process endpoint, a `formal_boundary` judgment with null value, and a separate `cycle_regime` or `no_formal_boundary_reason` judgment. This allows the existing view to identify the cycle and select pre-marker informal bids while keeping `formal_boundary` null. No-boundary cycles are valid observations, not validation errors.

Dropout-vs-screening is a first-class output. The canonical value set for `dropout_mechanism` is `target-rejected`, `voluntarily-withdrew`, `ambiguous`, and `n/a-admitted`. The value is not inferred mechanically from non-admission. It is a judgment based on linked withdrawal, rejection, notification, and silence events. Where the text says a party was told it was no longer in the process, the value is `target-rejected`. Where the text says a party withdrew, declined, became unable to proceed, or stopped responding before a target decision, the value is `voluntarily-withdrew` or `ambiguous` depending on clarity. Where the narrative does not reveal who moved first, the value is `ambiguous`, with an alternative value and basis if a plausible competing reading exists.

The estimator should not receive fabricated precision. Unknown bidder type, hidden individual bid values, incomplete unsolicited offers, mixed units, and rough dates remain visible through nulls and scope flags. The primary success condition is not that every row is estimation-ready; it is that every row is deterministically produced, evidence-backed, and marked so the estimator or analyst can apply a transparent inclusion rule.
