# Aggregation: Canonical Records to Bidder-Cycle Rows

Aggregation reconciles extraction candidates into canonical records and then projects estimator rows. It is deterministic and rule-based once the candidate set exists. Its outputs are canonical tables, validation flags, and derived views; it does not revisit raw text except through evidence spans.

Cycle construction comes first. Create a `process_cycles` record for each economically distinct sale process. A new cycle begins when the filing describes a process launch, restart, unsolicited single-bidder negotiation after a terminated process, or go-shop tail that the configuration treats as a separate cycle. A prior cycle ends when the process is terminated, abandoned, superseded by a new process, or signed into a merger agreement. Zep requires at least a 2014 terminated broad process and a 2015 New Mountain process; Saks and PetSmart each have one primary sale cycle plus a go-shop or post-signing tail to preserve.

Every cycle receives exactly one `formal_boundary` judgment. If the filing supports a real boundary, link the judgment to the transition event and set `value` to a concise label such as `final_round_invitation`, `confirmatory_diligence_shortlist`, `exclusive_confirmatory_diligence`, or `final_bid_request`. If the filing does not support a boundary, create a boundary-absence marker event at the best-supported cycle endpoint, set the judgment value to null, and add `cycle_regime = no_formal_boundary` or `terminated_before_formal_stage`. This keeps boundary absence observable while allowing pre-marker bids to be ranked.

Actor reconciliation merges aliases and assigns bidder subtypes. Strategic bidders are operating companies or industry participants, such as Hudson’s Bay or G&W. Financial bidders are sponsors or private equity firms, such as Sponsor A or New Mountain Capital. Mixed consortia receive `mixed_group` in the canonical table and a projection-specific `T` of null unless the analyst chooses a rule. Anonymous bidders created from aggregate counts receive `unknown` subtype unless the filing gives the strategic/financial split.

Proposal reconciliation assigns event eligibility. A proposal is `proposal_submitted` only if it belongs to the process and should be eligible for informal or formal bid selection. For example, PetSmart’s October 30 indications and December final bids are eligible. Saks’ Company H letter is preserved as `proposal_incomplete_unsolicited`, not as an estimator bid, because the filing says it lacked details and no contact followed. Providence and Worcester’s post-rejection revised LOIs from Party D and Party E are preserved as reengagement proposals unless the target actually readmitted the party to the confirmatory stage.

For each bidder-cycle, `bI` is the latest eligible proposal before the boundary date or boundary-absence marker. `bI_lo` and `bI_hi` come from the normalized bounds of that event. `bF` is the highest eligible proposal at or after a real boundary date. For no-boundary cycles, no event is classified as formal and `bF` remains null. `admitted` in the current view follows `derive_views.py`: it is true when `bF` exists. Separate canonical admission decisions are retained for future views.

Cycle statistics are computed from the bidder rows. `J` is the count of bidder actors in the cycle under the selected scope rule. The default research view counts all bidder rows with `scope_validity` not equal to `out_of_scope`, including anonymous aggregate actors. `g^S` is the share of those rows whose type is strategic, excluding unknown types from the denominator only if the view explicitly says so. `r_23` uses comparable non-null informal bid centers. `mean_w` uses rows with positive lower and upper bounds. `S` is true when `cycle_regime` indicates a real formal stage and at least one eligible formal proposal exists.

Dropout-vs-screening aggregation uses linked event evidence, not row absence. A bidder is `n/a-admitted` when it has an eligible formal proposal or an explicit canonical admission decision and no contrary withdrawal before formal bidding. It is `target-rejected` when the target, board, committee, or advisor eliminated it, did not invite it, or informed it that it was no longer involved. It is `voluntarily-withdrew` when the bidder declined, withdrew, failed to submit after being invited, or became unable to proceed before the target made an admission decision. It is `ambiguous` when the filing shows nonappearance but not causal order.

The strategy acknowledges fundamental ambiguity. If a filing says only that a party did not submit a final bid, aggregation cannot determine whether the target screened it out, the party lost interest, or both. The correct canonical judgment is `ambiguous`, with `alternative_value` populated when a plausible competing reading exists. This is not a pipeline failure; it is the signal the downstream robustness extension needs.

Aggregate counts are handled explicitly. When a filing reports six indications but names only three bidders, reconciliation creates named actors for identified bidders and anonymous actors for the remaining count if the count is necessary for `J`. Unknown individual bid values remain null and carry `scope_validity = incomplete_bid_value`. Aggregate bid ranges are stored as cycle-level count evidence, not falsely assigned to individual anonymous bidders.

The final projection writes the view fields expected by `derive_views.py` and companion review tables. Analysts can filter on `scope_validity`, `cycle_visibility`, `valuation_comparability`, and `confidence_min` before estimation. No projection is allowed to change source facts, invent missing bids, or resolve ambiguity without a recorded judgment.
